# Copy Frameworks — Biblioteca do Growth Copywriter

## Princípio Fundamental

**PRÉ-REQUISITO (copy de conversão):** Antes de escrever copy para landing page, sales deck ou e-mail de venda, verifique se o **Messaging Framework** (Seção 7 do Voice Guide do Bierut) está disponível. Se não estiver, solicite ao Bierut antes de prosseguir. Copy sem messaging framework tende a ser genérica e desalinhada com as dores reais de cada persona — o QA (Ive) reprovará.

Toda copy deve seguir esta sequência antes de ser escrita:

1. **Identifique o público** (qual persona do brief?)
2. **Consulte o Messaging Framework** dessa persona (mensagem-chave, proof points, objeção principal)
3. **Identifique o objetivo** (awareness? clique? conversão? retenção?)
4. **Identifique o canal** (ad? landing page? e-mail? social?)
5. **Escolha o framework** (baseado no objetivo e canal)
6. **Escreva** seguindo o framework
7. **Crie variações** para teste A/B
8. **Revise** contra os guardrails do brief

## Frameworks de Copy

### 1. AIDA — Attention, Interest, Desire, Action
**Melhor para:** Landing pages, e-mails de venda, anúncios longos

| Etapa | O que faz | Exemplo |
|-------|-----------|---------|
| **Attention** | Pare o scroll. Headline que gera curiosidade ou choque. | "87% dos e-commerces perdem vendas por este erro" |
| **Interest** | Conecte com a dor/desejo. Mostre que entende. | "Você investe em tráfego, mas o checkout afasta clientes..." |
| **Desire** | Mostre a transformação. Benefícios, não features. | "Nossos clientes aumentaram conversão em 34% em 60 dias" |
| **Action** | CTA claro e urgente. Uma ação, sem ambiguidade. | "Teste grátis por 14 dias → Comece agora" |

```
Template AIDA:

[HEADLINE — Attention]
________________________________

[SUBHEADLINE — Interest]
________________________________

[BODY — Desire]
- Benefício 1 (transformação, não feature)
- Benefício 2
- Prova social / dado concreto
________________________________

[CTA — Action]
________________________________
```

---

### 2. PAS — Problem, Agitation, Solution
**Melhor para:** Anúncios curtos, e-mails frios, posts sociais de conversão

| Etapa | O que faz | Dica |
|-------|-----------|------|
| **Problem** | Nomeie a dor do público. Seja específico. | Não diga "problemas de marketing". Diga "você gasta R$5k em ads e não sabe o que funciona" |
| **Agitation** | Amplifique a dor. Mostre as consequências. | "Enquanto isso, seu concorrente está roubando seus clientes..." |
| **Solution** | Apresente a marca como a saída. | "Com [Marca], você sabe exatamente onde cada real está gerando resultado" |

```
Template PAS:

[PROBLEM — 1-2 frases]
________________________________

[AGITATION — 1-2 frases que amplificam]
________________________________

[SOLUTION — Apresenta a marca + CTA]
________________________________
```

---

### 3. BAB — Before, After, Bridge
**Melhor para:** Storytelling em ads, e-mails de nutrição, posts de case study

| Etapa | O que faz |
|-------|-----------|
| **Before** | Pinte o cenário atual (com dor) |
| **After** | Pinte o cenário desejado (com prazer) |
| **Bridge** | A marca é a ponte entre os dois |

```
Template BAB:

[BEFORE — Como é a vida/situação ANTES]
________________________________

[AFTER — Como será DEPOIS]
________________________________

[BRIDGE — Como a marca leva do before ao after + CTA]
________________________________
```

---

### 4. 4Ps — Promise, Picture, Proof, Push
**Melhor para:** Landing pages longas, páginas de vendas, webinar pitches

| Etapa | O que faz |
|-------|-----------|
| **Promise** | Headline com a promessa principal (benefício claro) |
| **Picture** | Faça o leitor visualizar o resultado |
| **Proof** | Dados, testimonials, cases, logos de clientes |
| **Push** | CTA com urgência ou escassez |

---

### 5. ACCA — Awareness, Comprehension, Conviction, Action
**Melhor para:** Copy educativa, mercados com pouca consciência do problema

| Etapa | O que faz |
|-------|-----------|
| **Awareness** | Mostre que o problema existe |
| **Comprehension** | Explique o impacto do problema |
| **Conviction** | Convença que há solução (e que é você) |
| **Action** | CTA |

---

### 6. Star-Story-Solution
**Melhor para:** E-mails narrativos, posts de marca pessoal, vídeo scripts

| Etapa | O que faz |
|-------|-----------|
| **Star** | Apresente o protagonista (cliente ideal ou fundador) |
| **Story** | Conte a jornada (conflito → virada) |
| **Solution** | A marca como catalisador da transformação |

## Templates por Canal

### Anúncio Digital (Facebook/Instagram/Google)

```
FORMATO CURTO (até 125 caracteres — primary text):
[Hook de 1 linha] + [Benefício] + [CTA]

FORMATO MÉDIO (até 250 caracteres):
[Problem — 1 frase]
[Agitation — 1 frase]
[Solution + CTA]

FORMATO LONGO (até 600 caracteres):
[Framework completo PAS ou AIDA]
```

**Regras para ads:**
- Headline: máx 40 caracteres (Facebook), 30 caracteres (Google)
- Primeira linha é TUDO — decide se o usuário lê o resto
- Números concretos > Afirmações vagas ("34% mais rápido" > "muito mais rápido")
- Uma única CTA por anúncio
- Sempre criar mínimo 3 variações para teste A/B

### Landing Page

```
ESTRUTURA PADRÃO:
1. HERO: Headline (promessa) + Subheadline (como) + CTA primário
2. PROVA SOCIAL: Logos, números, depoimento curto
3. PROBLEMA: Nomeie a dor (PAS)
4. SOLUÇÃO: Como a marca resolve (features → benefits)
5. BENEFÍCIOS: 3-4 blocos com ícone + título + descrição
6. PROVA: Cases, testimonials detalhados, dados
7. FAQ: Objeções disfarçadas de perguntas
8. CTA FINAL: Repetir CTA com urgência
```

### E-mail Marketing

```
SUBJECT LINE:
- Máx 50 caracteres (idealmente 30-40)
- Personalização quando possível
- Criar curiosidade OU prometer benefício
- Nunca ALL CAPS, nunca excesso de emoji
- Sempre criar 2-3 variações para teste A/B

BODY:
- Primeira frase: hook ou pergunta
- Corpo: Use BAB ou PAS (e-mails curtos) ou AIDA (e-mails de venda)
- Um único CTA (botão ou link)
- P.S.: Repetir oferta ou adicionar urgência (alta taxa de leitura)
```

### Post Social (Orgânico)

```
HOOK (primeira linha):
- Pergunta provocativa
- Dado surpreendente
- Afirmação contraintuitiva
- "A maioria das pessoas..."

BODY:
- Conteúdo de valor (use framework adequado ao objetivo)
- Parágrafos curtos (1-2 linhas)
- Espaço entre parágrafos

CTA:
- Engajamento: "Comenta aqui se..."
- Tráfego: "Link na bio"
- Salvar: "Salva pra consultar depois"
```

## Processo de Teste A/B

Para cada peça de copy, o Ogilvy DEVE criar no mínimo 2 variações:

### O que testar (em ordem de impacto):
1. **Headlines** (maior impacto em conversão)
2. **CTAs** (segundo maior impacto)
3. **Abordagem/Framework** (ex: PAS vs AIDA)
4. **Comprimento** (curto vs longo)
5. **Tom** (mais formal vs mais casual)

### Formato de entrega de variações:

```markdown
## [Canal] — [Objetivo]

### Variação A: [Racional — ex: "Foco em dor"]
[Copy completa]

### Variação B: [Racional — ex: "Foco em aspiração"]
[Copy completa]

### Variação C (opcional): [Racional — ex: "Foco em prova social"]
[Copy completa]

### Hipótese de teste:
"Acreditamos que [Variação X] terá melhor performance porque [razão] para [persona]."

### Métrica primária de sucesso:
[CTR / Conversão / Abertura / etc.]
```

## Checklist de Qualidade — Copy Review

Antes de entregar qualquer copy, validar:

- [ ] O Messaging Framework da persona-alvo foi consultado? (obrigatório para copy de conversão)
- [ ] Está alinhada com o Brand Voice Guide (Bierut)?
- [ ] Respeita os Strategic Guardrails do brief (Neumeier)?
- [ ] O público-alvo reconheceria sua dor/desejo na primeira linha?
- [ ] Há um único CTA claro?
- [ ] Os benefícios estão em linguagem do cliente (não jargão interno)?
- [ ] Há prova social ou dado concreto (proof point verificável)?
- [ ] Foram criadas variações para A/B testing?
- [ ] O comprimento é adequado ao canal?
- [ ] Lida em voz alta, soa natural?
