# Brand QA Checklists — Guia do Brand Experience Curator

## Princípio

O Ive é o último filtro antes de qualquer entrega ir ao público. Seu papel não é subjetivo — é sistemático. Cada tipo de entrega tem critérios objetivos de aprovação. Se um item falha, a entrega volta ao agente responsável com feedback específico e acionável.

## Processo de QA

```
1. Receber entrega do agente
2. Identificar o tipo de entrega
3. Aplicar o checklist correspondente
4. Classificar cada item: ✅ Aprovado | ⚠️ Precisa ajuste | ❌ Reprovado
5. Se todos ✅ → Aprovado para publicação
6. Se algum ⚠️ → Aprovado com ressalvas (listar ajustes menores)
7. Se algum ❌ → Devolver ao agente com feedback específico
```

---

## Checklist Universal (aplica-se a TUDO)

Estes critérios se aplicam a qualquer entrega, independentemente do tipo:

| # | Critério | Verificação |
|---|----------|-------------|
| U1 | **Alinhamento estratégico** | A entrega reflete o ZAG e o Golden Circle do brief? |
| U2 | **Consistência visual** | Cores, tipografia e logo estão conforme o Brand Book? |
| U3 | **Consistência verbal** | Tom de voz, vocabulário e estilo conforme o Voice Guide? |
| U4 | **Público-alvo** | A entrega fala para a persona correta? |
| U5 | **Guardrails** | Nenhum guardrail do brief foi violado? |
| U6 | **Qualidade textual** | Sem erros de ortografia, gramática ou formatação? |
| U7 | **Acessibilidade básica** | Contraste legível, hierarquia clara, tamanho adequado? |
| U8 | **Acessibilidade de cor** | Cores de acento usadas em texto pequeno passam WCAG AA 4.5:1? (Verificar especialmente success/error/warning — versões escuras obrigatórias para badges e labels) |

## Checklists por Tipo de Entrega

### 📋 Brand Strategy Brief (do Neumeier)

| # | Critério | Verificação |
|---|----------|-------------|
| B1 | **Completude** | Todas as 11 seções do template estão preenchidas? |
| B2 | **Golden Circle** | O WHY é inspirador e não genérico? |
| B3 | **ZAG Statement** | É impossível aplicar a um concorrente? |
| B4 | **Personas** | Têm profundidade suficiente (dores, desejos, objeções, canais)? |
| B5 | **KPIs** | Os objetivos têm métricas mensuráveis e prazos? |
| B6 | **Guardrails** | São concretos e específicos (não vagos)? |
| B7 | **Acionabilidade** | Cada agente consegue iniciar seu trabalho a partir do brief? |
| B8 | **Tempo de leitura** | Pode ser lido e compreendido em 10-15 minutos? |

**Critério de aprovação:** Todos B1-B7 devem ser ✅. B8 é ⚠️ aceitável.

---

### 🖋️ Logotipo (do Haviv)

| # | Critério | Verificação |
|---|----------|-------------|
| L1 | **Filtro Apropriado** | O símbolo é adequado ao setor e ao público? |
| L2 | **Filtro Distinto** | É claramente diferente dos concorrentes? |
| L3 | **Filtro Simples** | Pode ser descrito em 1-2 frases? Funciona em monocromia? |
| L4 | **Escalabilidade** | Funciona de 16x16px a outdoor? |
| L5 | **Versatilidade** | Funciona em fundo claro, escuro e sobre imagem? |
| L6 | **Rationale** | Cada escolha de forma e cor é justificada pelo brief? |
| L7 | **Originalidade** | Não se assemelha a marcas existentes conhecidas? |
| L8 | **Spec executável** | Há especificação tipográfica/geométrica suficiente para o Bauhaus reproduzir sem o Haviv presente? (fonte exata, peso, cor hex, grid de símbolo) |

**Critério de aprovação:** L1-L5 e L8 devem ser ✅. L6-L7 são ⚠️ aceitáveis com justificativa.

---

### 🧬 Brand Book / Sistema Visual (do Scher)

| # | Critério | Verificação |
|---|----------|-------------|
| S1 | **Paleta de cores** | Primárias, secundárias e terciárias definidas com códigos (HEX/RGB/CMYK/Pantone)? |
| S1a | **Acessibilidade de paleta** | Cores de acento verificadas para WCAG AA? Versões escuras para texto pequeno documentadas? |
| S2 | **Tipografia** | Famílias, pesos, tamanhos e hierarquia documentados? |
| S3 | **Grid system** | Princípios de layout com exemplos de aplicação? |
| S4 | **Elementos gráficos** | Patterns, ícones ou ilustrações definidos? |
| S5 | **Diretrizes de imagem** | Estilo fotográfico, tratamento e do's/don'ts? |
| S6 | **Logo usage** | Área de proteção, versões, usos incorretos? |
| S7 | **Aplicações** | Mínimo 8 mockups — incluindo pelo menos 4 contextos externos ao produto (landing page, social, e-mail, slide de vendas)? |
| S8 | **Flexibilidade** | O sistema permite variação sem perder identidade? |
| S9 | **Praticabilidade** | Um designer consegue aplicar as regras sem ambiguidade? |

**Critério de aprovação:** S1-S6 devem ser ✅. S7-S9 são ⚠️ aceitáveis.

---

### 📖 Brand Voice Guide (do Bierut)

| # | Critério | Verificação |
|---|----------|-------------|
| V1 | **Personalidade** | 3-5 traços com espectro "Somos X, não Y"? |
| V2 | **Tom de voz** | Espectros definidos (formal↔casual, etc.)? |
| V3 | **Vocabulário** | Palavras que usamos vs. evitamos? |
| V4 | **Exemplos por canal** | Como a voz se adapta em pelo menos 3 canais? |
| V5 | **Manifesto** | Manifesto da marca escrito e revisado? |
| V6 | **Taglines** | Pelo menos 1 tagline principal + variações? |
| V7 | **Messaging framework** | Para cada persona: mensagem-chave + 3 proof points + resposta à objeção principal documentados? |
| V8 | **Aplicabilidade** | O Ogilvy e Godin conseguem escrever usando apenas este guia? |

**Critério de aprovação:** V1-V5 e V7 devem ser ✅. V6, V8 são ⚠️ aceitáveis.

**Nota V7:** O messaging framework é pré-requisito para qualquer copy de landing page ou sales deck. Não aprovar o Voice Guide sem ele — devolver ao Bierut.

### ✍️ Copy de Campanha (do Ogilvy)

| # | Critério | Verificação |
|---|----------|-------------|
| C1 | **Framework** | Usa um framework identificado (AIDA, PAS, BAB, etc.)? |
| C2 | **Headline** | Gera curiosidade OU promete benefício claro? |
| C3 | **CTA** | Único, claro, e com verbo de ação? |
| C4 | **Persona** | Fala a linguagem da persona-alvo? |
| C5 | **Benefícios vs Features** | Foca em transformação, não em especificações? |
| C6 | **Variações A/B** | Mínimo 2 variações com hipótese de teste? |
| C7 | **Comprimento** | Adequado ao canal (respeita limites de caracteres)? |
| C8 | **Brand Voice** | Consistente com o Voice Guide do Bierut? |
| C9 | **Prova** | Inclui dado concreto, testimonial ou prova social? |

**Critério de aprovação:** C1-C5 e C8 devem ser ✅. C6-C7, C9 são ⚠️ aceitáveis.

### 📝 Conteúdo (do Godin)

| # | Critério | Verificação |
|---|----------|-------------|
| CT1 | **Valor** | O conteúdo ensina, inspira ou entretém o público? |
| CT2 | **SEO** | Keywords relevantes incorporadas naturalmente? |
| CT3 | **Funil** | Estágio do funil está identificado (topo, meio, fundo)? |
| CT4 | **CTA** | Tem próximo passo claro (mesmo que suave)? |
| CT5 | **Brand Voice** | Consistente com o Voice Guide do Bierut? |
| CT6 | **Originalidade** | Traz perspectiva única, não repete o óbvio? |
| CT7 | **Formatação** | Adequada ao canal (blog, social, vídeo, etc.)? |
| CT8 | **Calendário** | Está no calendário editorial com data e responsável? |

**Critério de aprovação:** CT1, CT5-CT6 devem ser ✅. Resto é ⚠️ aceitável.

### 🎨 Assets Visuais (do Bauhaus)

| # | Critério | Verificação |
|---|----------|-------------|
| A1 | **Brand Book** | Segue rigorosamente cores, tipografia, grid do Scher? |
| A2 | **Especificações** | Dimensões corretas para o canal/plataforma? |
| A3 | **Hierarquia** | Informação mais importante é a mais visível? |
| A4 | **Legibilidade** | Texto legível em todos os tamanhos de exibição? |
| A5 | **Copy** | A copy do Ogilvy/Godin foi integrada corretamente? |
| A6 | **Qualidade técnica** | Resolução adequada, sem pixelização, cores corretas? |
| A7 | **Variações** | Versões para diferentes formatos quando necessário? |

**Critério de aprovação:** A1-A3 e A5-A6 devem ser ✅. A4, A7 são ⚠️ aceitáveis.

### 📊 Plano de Marketing (do Kotler)

| # | Critério | Verificação |
|---|----------|-------------|
| M1 | **KPIs** | Métricas são SMART (específicas, mensuráveis, atingíveis, relevantes, temporais)? |
| M2 | **Canais** | Seleção de canais é justificada pelo perfil das personas? |
| M3 | **Orçamento** | Alocação por canal está definida e justificada? |
| M4 | **Timeline** | Cronograma de execução com milestones? |
| M5 | **Segmentação** | Públicos de mídia alinhados com personas do brief? |
| M6 | **Tracking** | Estrutura de tracking e atribuição está definida? |
| M7 | **Cenários** | Há projeção de cenários (otimista/realista/pessimista)? |

**Critério de aprovação:** M1-M4 devem ser ✅. M5-M7 são ⚠️ aceitáveis.

## Formato do Relatório de QA

Quando o Ive auditar uma entrega, o relatório deve seguir este formato:

```markdown
# QA Report — [Tipo de Entrega]
## Agente: [Nome]
## Data: [DD/MM/AAAA]

### Resultado: ✅ Aprovado | ⚠️ Aprovado com ressalvas | ❌ Devolvido

### Detalhamento:

| # | Critério | Status | Comentário |
|---|----------|--------|------------|
| U1 | Alinhamento estratégico | ✅ | — |
| U2 | Consistência visual | ⚠️ | Cor secundária fora do padrão no banner 3 |
| ... | ... | ... | ... |

### Itens para correção:
1. [Item específico] → [Ação necessária] → [Agente responsável]
2. ...

### Próximo passo:
- [ ] Agente corrige e reenvia
- [ ] Ive faz re-review
```

## Experience Audit (Auditoria Completa)

Para auditorias periódicas de toda a experiência da marca:

### Touchpoints a auditar:
1. **Digital**: Site, app, redes sociais, e-mail, ads
2. **Conteúdo**: Blog, vídeos, podcasts, e-books
3. **Comunicação**: E-mails transacionais, atendimento, chatbot
4. **Físico** (se aplicável): Loja, embalagem, papelaria, eventos
5. **Interno**: Cultura, onboarding, comunicação interna

### Para cada touchpoint, avaliar:
- Consistência visual (1-5)
- Consistência verbal (1-5)
- Alinhamento com ZAG/propósito (1-5)
- Qualidade da experiência (1-5)
- Nota geral do touchpoint

### Entrega: Brand Experience Scorecard
Matriz com todos os touchpoints, notas, e plano de ação para os itens abaixo de 4.
